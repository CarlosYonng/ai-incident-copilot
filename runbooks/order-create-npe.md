# 订单创建空指针 Runbook

## 适用场景

- `/api/orders` 创建订单时报 `NullPointerException`。
- 日志出现 `userProfile is null when creating order`。
- 用户下单失败或订单创建接口 500。

## 常见症状

- `order-service` 500 错误率升高。
- 新用户或部分用户创建订单失败。
- trace 指向用户资料查询或订单组装逻辑。

## 排查步骤

1. 查询 `order-service` 错误日志。
2. 查询代码中 `userProfile` 使用位置。
3. 检查 `user-service` 返回是否为空。
4. 查询近期是否修改订单创建或用户资料字段。
5. 查询历史类似空指针工单。

## 常见原因

- `user-service` 返回空对象。
- 订单创建逻辑缺少空值兜底。
- 新字段上线后兼容逻辑不足。
- 测试用例没有覆盖用户资料缺失场景。

## 低风险动作

- 创建后续修复任务。
- 补充单元测试。
- 补充日志字段。
- 通知订单服务负责人。

## 中风险动作

- 临时启用用户资料缺失兜底逻辑。
- 对异常用户流量进行灰度降级。

## 高风险动作

- 回滚订单服务版本。
- 修改生产数据库中的用户资料数据。

## 前置检查

- 确认空值兜底不会产生错误订单。
- 确认下游库存、支付、优惠券链路可接受默认用户资料。

## 观察指标

- `/api/orders` 500 错误率。
- 创建订单成功率。
- `user-service` 空响应数量。

## 复盘要求

- 补充 NPE 防护规范。
- 增加 userProfile 缺失测试用例。
- 明确用户服务契约和降级策略。
