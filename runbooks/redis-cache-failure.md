# Redis 缓存故障 Runbook

## 适用场景

- Redis 连接超时或不可用。
- 缓存命中率骤降。
- 缓存击穿或雪崩导致数据库压力升高。

## 常见症状

- 日志出现 Redis timeout、connection reset。
- 数据库 QPS 升高。
- 接口延迟升高。
- 缓存命中率下降。

## 排查步骤

1. 查看 Redis 实例状态和连接数。
2. 检查缓存命中率和 key 过期情况。
3. 查看数据库压力是否被放大。
4. 检查是否有大 key 或热 key。
5. 查询近期缓存策略变更。

## 常见原因

- Redis 实例异常。
- 热 key 导致单点压力。
- 大量 key 同时过期。
- 缓存穿透或击穿。

## 低风险动作

- 通知缓存负责人。
- 查询热 key 和大 key。
- 补充监控。

## 中风险动作

- 临时开启本地缓存兜底。
- 对高风险入口限流。
- 调整非核心缓存过期策略。

## 高风险动作

- 重启 Redis。
- 修改生产 Redis 配置。
- 批量删除生产 key。

## 前置检查

- 确认本地缓存不会造成严重数据不一致。
- 确认限流策略对用户影响可接受。

## 观察指标

- Redis 可用性。
- 缓存命中率。
- 数据库 QPS。
- 接口 p95 延迟。

## 复盘要求

- 说明缓存故障是否引发数据库压力放大。
- 补充热 key、大 key 和缓存过期治理计划。
