# 数据库慢查询 Runbook

## 适用场景

- 接口延迟升高并伴随数据库慢查询。
- 数据库连接池耗尽。
- 日志出现 SQL timeout 或 lock wait timeout。

## 常见症状

- p95 / p99 延迟显著升高。
- 数据库 CPU 或 IO 升高。
- 慢查询数量增加。
- 连接池 active 连接接近上限。

## 排查步骤

1. 查询慢 SQL 和执行计划。
2. 查看数据库 CPU、IO、连接数和锁等待。
3. 检查近期 SQL、索引或数据量变化。
4. 确认是否有批处理或报表任务挤占资源。
5. 查询历史慢查询工单。

## 常见原因

- 缺失索引。
- SQL 条件选择性差。
- 大事务持锁。
- 突发批处理任务。

## 低风险动作

- 通知 DBA 或数据库负责人。
- 暂停非关键报表观察。
- 创建 SQL 优化任务。

## 中风险动作

- 临时关闭非核心查询入口。
- 限流高成本接口。

## 高风险动作

- 在线新增索引。
- Kill 生产会话。
- 修改数据库参数。
- 执行生产 SQL。

## 前置检查

- 确认限流不会影响核心交易。
- 确认索引变更经过 DBA 审核。

## 观察指标

- 慢查询数量。
- 数据库 CPU 和 IO。
- 连接池 active / idle。
- 接口 p95 延迟。

## 复盘要求

- 给出 SQL、执行计划和索引建议。
- 增加慢查询监控和容量评估。
