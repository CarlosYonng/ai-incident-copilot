# 下游依赖不可用 Runbook

## 适用场景

- 服务调用下游 HTTP / RPC 依赖超时。
- 下游返回 5xx。
- 连接池耗尽或熔断器打开。

## 常见症状

- 上游 p95 延迟升高。
- 日志出现 timeout、connection refused、circuit open。
- QPS 未明显上升但错误率升高。

## 排查步骤

1. 确认异常依赖名称和接口。
2. 查看下游服务错误率、延迟和实例状态。
3. 检查网络、DNS、连接池和熔断状态。
4. 查询最近发布和配置变更。
5. 检查是否存在重试风暴。

## 常见原因

- 下游服务实例异常。
- 网络或 DNS 问题。
- 连接池配置不足。
- 重试策略放大流量。

## 低风险动作

- 通知下游负责人。
- 开启只读观察。
- 创建后续任务。

## 中风险动作

- 临时降级非核心依赖。
- 限流异常入口。
- 调整调用超时时间。

## 高风险动作

- 重启下游服务。
- 回滚下游版本。
- 修改生产配置。

## 前置检查

- 确认降级不会影响核心交易一致性。
- 确认限流策略对用户影响可接受。

## 观察指标

- 上游错误率。
- 下游错误率。
- 熔断状态。
- 连接池使用率。

## 复盘要求

- 说明依赖不可用根因。
- 检查是否需要增加熔断、降级和超时隔离。
