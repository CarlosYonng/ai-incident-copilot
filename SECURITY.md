# 安全策略

## 支持的版本

本项目目前处于 MVP / RC（候选发布）阶段。在建立稳定的发布流程之前，安全修复仅应用于 `main` 分支。

## 安全边界

AI Incident Copilot 被设计为一个以只读为主的**故障协作系统**，**不会**执行生产环境的修复操作。

- 系统不会重启服务器。
- 系统不会执行生产环境的 SQL。
- 系统不会修改生产环境的配置。
- 系统不会回滚部署。
- 中、高风险操作需要人工确认，并被记录为审计事件。

## 密钥管理

`.env.example` 和 `docker-compose.yml` 中的默认值仅供本地演示使用。在任何共享或部署环境中：

- 请替换所有 MySQL 密码。
- 如果 Diagnosis MCP 暴露在非 localhost 环境，请配置 `DIAGNOSIS_MCP_TOKEN`。
- 请限制 `CORS_ALLOWED_ORIGINS`。
- 不要公开暴露 MySQL 端口。

## 问题上报

如果发现安全问题，请在公开 issue 中**不要包含**真实的密钥、token、数据库 dump 或真实故障数据。请通过私有渠道提交报告或直接联系仓库所有者。
